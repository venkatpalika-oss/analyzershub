ABB GC Alarm Families (Practical Codes Table)

Use this when you see an alarm message like “Pressure low”, “Oven not ready”, “Valve sequence”, “Communication fail”, etc.

Alarm family What you see Most likely causes First checks (5–10 min) Corrective actions
Carrier / Pressure Low carrier pressure, unstable baseline, no peaks Empty cylinder, regulator hunting/creep, leaks, restriction
  • Carrier cylinder pressure OK
  • Regulator outlet stable
  • Leak test critical joints
  • Check filters/orifice
 Restore carrier supply, repair leaks, replace/clean restriction, rerun 2–3 cycles to confirm.
Decision tree →
Sampling / Flow Weak peaks, slow response, unstable results Plugged filter, liquid carryover, unstable sample pressure/flow
  • Check sample pressure stability
  • Inspect traps for liquids
  • Check filters/coalescer
  • Check stream selection
 Drain/replace filters, restore stable flow, improve conditioning. Validate with repeatable chromatograms.
Chromatogram guide →
Oven / Temperature Oven not ready, heater fault, temperature unstable Heater power issue, sensor/wiring, ventilation, controller fault
  • Setpoint = actual?
  • Check fuses/breakers
  • Inspect sensor wiring
  • Check cabinet ventilation
 Restore power path, fix sensor wiring, clear airflow. Run a stability hold then test.
Decision tree →
Valve / Sequence Injection fail, sequence error, valve not switching Dirty/wet instrument air, solenoid fault, mechanical sticking
  • Instrument air dry/oil-free
  • Observe switching/solenoid LED
  • Check coil wiring/loose connections
 Restore clean air, service solenoid/valve, rerun cycle.
Decision tree →
Detector / Electronics Noise, drift, unstable baseline, detector faults Grounding, contamination, utility instability, electronics issue
  • Confirm stable carrier & oven first
  • Check grounding/shielding
  • Review recent maintenance
 Stabilize utilities, address grounding/noise source, clean contamination sources. Verify with repeated stable runs.
Communication / DCS No update, bad quality, comms fail IP/network issues, mapping, service down, cycle timing misunderstood
  • Check link LEDs + ping test
  • Verify IP/subnet/gateway
  • Confirm protocol/service running
  • Confirm cycle time expectation
 Fix network/mapping, confirm values update after each completed cycle.
Decision tree →
Field best practice: Always verify with 2–3 consecutive stable cycles before releasing values to operations.
