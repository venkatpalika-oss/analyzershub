Oven Temperature Fault — Decision Tree

Use when: oven not ready, temperature unstable, over/under temp, slow ramp, heater/sensor issues.

Safety: High temperature components. Follow electrical isolation rules before touching heater/sensor wiring.
Q1 — Is the oven temperature close to setpoint (within acceptable tolerance)?
  • YES: Go to Q2.
  • NO: Go to Q3.
Q2 — Is temperature stable (no oscillation / no sudden jumps)?
  • YES: Alarm may be historical or intermittent. Check alarm log time → clear/acknowledge → monitor 30 minutes.
  • NO: Suspect sensor drift, loose wiring, door/gasket leak, controller tuning → go to Q4.
Q3 — Is the oven heater receiving power (as per diagnostic/status)?
  • YES: Go to Q4.
  • NO: Check fuses/breakers, interlocks, heater relay/SSR, wiring integrity → restore safely → recheck.
Q4 — Is the temperature sensor reading reasonable (not open/short / extreme value)?
  • YES: Go to Q5.
  • NO: Sensor wiring open/short or sensor failed → repair wiring / replace sensor → recheck stability.
Q5 — Is there airflow or enclosure issue causing heat loss?
  • YES: Check oven door seal/gasket, cabinet vents, fan issues, drafts → correct → recheck ramp and stability.
  • NO: Go to Q6.
Q6 — Does the method require a specific ramp profile/temperature?
  • YES: Confirm method parameters match the validated configuration (backup/last known good).
  • NO / Unknown: Compare with last known good method or restore from backup.
Verification:
  • Oven reaches setpoint and remains stable for 15–20 minutes.
  • Retention times return to normal range.
  • Repeat run shows consistent peak shape and RT.
If still failing:
  • Capture alarm text + temperature trend screenshot.
  • Record setpoint vs actual, heater output status, and sensor reading.
  • Escalate: suspected heater element failure, control board fault, or sensor circuit issue.
