Decision Tree: Communication / DCS

← Back to Alarms Hub Alarm Codes → Back to SITRANS CV
Goal: stable updates to DCS/PLC (CV/Wobbe values + quality + correct timing).
1) Confirm analyzer is cycling and timestamps update normally.
2) Verify physical/network basics: link lights, IP/subnet/gateway, ping test.
3) Confirm protocol settings: Modbus TCP / OPC / serial gateway match DCS config.
4) Verify mapping/scaling: CV/Wobbe tags, data types, scaling, and quality/status mapping.
5) Align DCS polling with GC cycle time (avoid timeouts or “stale data”).

Quick Symptoms → Likely Cause

All values stale: IP/network, service down, wrong port/protocol.

Values update but wrong: mapping offset/scaling/data type mismatch.

Bad quality only: quality mapping or polling mismatch with cycle time.
