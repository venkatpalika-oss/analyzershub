Decision Tree: Communication / DCS

← Back to Alarms Hub Alarm Codes → Back to Analyzer
Goal: stable DCS updates (value + quality + correct scaling).
1) Confirm Wobbe value is stable locally first (trend is steady).
2) Check network: link lights, correct IP/subnet/gateway, ping test.
3) Confirm protocol settings (Modbus/OPC/etc.) match DCS.
4) Verify mapping/scaling/data types + quality/status mapping.
5) Confirm polling interval matches analyzer update rate.

Major Troubleshooting

  • Stale values: network/protocol service down, wrong port, wrong mapping.
  • Wrong values: scaling mismatch or wrong register offset/data type.
  • Bad quality: quality mapping/polling mismatch.
